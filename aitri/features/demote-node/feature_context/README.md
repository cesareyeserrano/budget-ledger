# feature_context/ — provided definitions for this feature

Drop the material specific to THIS feature here: a spec, business rules, mockups, PDFs, screenshots, sample code.
At the requirements/design phases Aitri injects the full text of the readable files here into the briefing, and flags what it cannot read (images, oversized files) to be opened — so provided definitions reach the agent, not just their filenames.

Notes:
- This is the feature's own context. The root project has its own `idea_context/` (surfaced to the feature as a read-only pointer).
- Keep it curated: remove material that becomes obsolete as the feature evolves.
