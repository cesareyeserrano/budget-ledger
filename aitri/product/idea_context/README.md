# idea_context/ — supporting assets

Drop reference material here: mockups, Figma exports, PDFs, a prior PRD, screenshots, reference docs.
Aitri automatically lists the files in this folder in every phase briefing so your agent can reference them.

Notes:
- This folder (`aitri/product/idea_context/`) is for supporting assets. The seed brief you write is the `aitri/product/IDEA.md` file — a different thing.
- The approved artifacts (in aitri/product/spec/) are the source of truth. Keep this folder curated: remove material that becomes obsolete as the project evolves, so it does not mislead the agent.
